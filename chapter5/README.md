## Structural Design Patterns


1. Concerned with how classes and objects are composed to form larger structures. 

2. Structural design patterns use inheritance to compose interfaces and define ways to compose objects to obtain new functionality.


### Example of Structural Design Pattern: 

1. Adapter Design Pattern: makes an interface to conform to each other. it makes a uniform of abstractions of different interfaces.

2. Composite Design Pattern : describe how to build class hierarchies made up of classes for two kinds of objects: primitive and composite. It allows clients to treat individual objects and compositions of objects uniformly.

3. Proxy Design Pattern: provides a surrogate or placeholder for another object to control access to it.

4. Flyweight Design Pattern: defines a structure for sharing objects

5. Decorator design pattern: how to add responsibilities to objects dynamically. It provides a flexible alternative to subclassing for extending functionality.

6. Facade design pattern: how to single object represents an entire subsystem. It provides a unified interface to a set of interfaces in a subsystem. Facade defines a higher-level interface that makes the subsystem easier to use.

7. Bridge design pattern: separates an objects abstraction from its implementation so that we can vary the abstraction and implementation independently. It decouples an abstraction from its implementation so that the two can vary independently.
